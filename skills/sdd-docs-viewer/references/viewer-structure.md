# Viewer Structure Reference

## Goal

Provide a repeatable browser-readable shell for Markdown docs persisted under `docs/`.

Those docs may not exist yet in a fresh analyzed project. The viewer must tolerate that empty state.

The reusable source for that shell lives in `skills/sdd-docs-viewer/`.

The preferred model is:

- viewer implementation inside the skill
- project docs persisted under `docs/`
- exported standalone site folder generated by default for persisted docs, while remaining a derived artifact

## Source of Truth

Keep the reusable implementation in:

- `skills/sdd-docs-viewer/assets/`
- `skills/sdd-docs-viewer/scripts/`

Do not depend on `docs/site/` as the primary home of the viewer.

If a standalone site export is needed, generate it from the skill. Do not maintain the exported copy as the source of truth.
When docs are being persisted for developer consumption, treat `docs/site/` as the default export target unless the user explicitly opts out.

## Layout Contract

The viewer exposes:

1. A sidebar with document search and grouped navigation (collapsible on mobile)
2. A content area for rendered Markdown
3. Mermaid rendering inside the content area
4. A content header showing current document title, category, last modified date, and estimated read time
5. An auto-generated table of contents from document headings (h2, h3)
6. Keyboard navigation between documents (arrow keys)
7. A back-to-top floating button
8. Hash-based deep linking to specific headings

No theme toggles, no pan-zoom controls.

## Manifest Contract

The skill-local `assets/manifest.json` should contain:

- `generatedAt`
- `docs[]`

Each item in `docs[]` should include:

- `slug`
- `title`
- `path`
- `category`
- optional `description`
- optional `lastModified` (ISO date from file modification time)
- optional `wordCount` (approximate word count for read-time estimation)

## Document Discovery Rules

Include:

- `docs/README.md`
- `docs/flows/README.md`
- `docs/flows/*.md`
- nested flow docs when they exist

If `docs/` does not exist yet, discovery should return an empty docs list rather than fail.

Exclude:

- `docs/flows/diagrams/**`
- binary assets

An exported site folder, if one exists, should also be excluded.

## UX Contract

Keep the shell readable and responsive:

- sticky sidebar on desktop
- collapsible sidebar on narrow screens (< 768px) with hamburger toggle
- active document highlight
- graceful empty state when no docs exist yet
- clear error state if a referenced Markdown file cannot be loaded
- auto-generated table of contents for documents with 3+ headings
- smooth scroll-to-heading on TOC link click
- print styles that hide sidebar and navigation for clean PDF/print output
- focus-visible styles for keyboard accessibility
